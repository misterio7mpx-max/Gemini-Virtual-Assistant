📖 概要 (Overview)
ブラウザ上のAIチャット（Gemini）を、音声認識・音声合成・リップシンク（口パク）を備えた「完全なバーチャルアシスタント」へと拡張するTampermonkey向けユーザースクリプトです。
テキストベースのUIとは独立したアバターウィンドウ上で、ハンズフリーの音声対話を実現します。

✨ 主な機能 (Features)
🗣️ ローカル音声合成連携 (VOICEVOX対応)
ローカル環境（localhost:50021）のAPIと通信し、AIの生成テキストをリアルタイムに音声キューとして処理・再生します。

👄 音声解析によるリアルタイム・リップシンク
AudioContext と AnalyserNode を用いて再生中の音声周波数を解析し、音量に合わせてアバターの口パクを動的に制御します。

🖼️ 独立したUIウィンドウ
ドラッグやリサイズが可能な専用の「アバター・コントロールウィンドウ」を生成し、デスクトップ上に自由に配置可能です。

⚙️ 高度なステートマシンとDOM監視
MutationObserver でAIのストリーミング出力を監視。「待機」「思考中」「発話中」「聞き取り中」などのステータスを管理し、状態に応じたアニメーションやタイピング音を自動で切り替えます。

🎤 ハンズフリー音声対話
Web Speech APIを利用した音声認識機能を搭載。ウェイクワードによる起動、スリープワードへの移行、追記や送信の制御に柔軟に対応しています。

🧩 各モジュールの役割 (Modules)
システムは以下のクラスに分割され、連携して動作します。

VoiceSettings: ユーザー設定（各種ワード、画像パス、音声パラメータ）の管理とローカルストレージへの保存を行います。

VoiceState: システムの現在状態（待機、思考、発話など）や、タイマーの数値を一括で管理します。

VoiceAudio: Web Audio APIを使用し、マイク入力の取得、リップシンク用の波形解析、および各種システム音の再生を担当します。

VoiceTTS: VOICEVOX APIと通信し、音声データの生成、キャッシュ処理、およびキューイングによる滑らかな連続再生を行います。

VoiceSpeech: 音声認識エンジンです。設定されたワード（完全一致・部分一致）を判定し、Geminiへのテキスト入力や状態切り替えをトリガーします。

VoiceImageCache: 外部の画像データをBlob URLとしてメモリにキャッシュし、ブラウザのセキュリティ制限（CSP）を安全に回避して画像を描画します。

VoiceUI: コントロールパネルの構築、ドラッグ＆リサイズ制御、ステータステキストやプログレスバーの描画、アバターのアニメーションを管理します。

VoiceObserver: Geminiの画面（DOM）の変化を監視し、AIの返答開始・終了の検知や、読み上げ用テキストの抽出を行います。

VoiceApp: 上記すべてのモジュールを統合し、システムの起動シーケンスを管理するメインコントローラーです。

🛠️ 技術的なアプローチ (Technical Approach)
リソース（アセット）の外部化
初期開発において、アバター画像等のBase64エンコードによるスクリプト内包を試みましたが、データ容量の肥大化によるブラウザのメモリ圧迫を回避するため、アーキテクチャを見直しました。
現在、UI・ロジック（本スクリプト）と画像リソース（外部URL参照）を分離する設計を採用し、さらに VoiceImageCache モジュールによるBlob変換を挟むことで、軽量かつセキュアな動作を図っています。

💻 動作環境 (Requirements)
Tampermonkey などのユーザースクリプトマネージャー

マイク入力が許可されたブラウザ環境

ローカル音声合成エンジン: VOICEVOX等のAPIサーバー（デフォルト設定: localhost:50021）

ローカルWebサーバー: 画像リソースを配信するためのサーバー（例: Pythonの python -m http.server 8000 など。デフォルト設定: http://localhost:8000/）

🚀 インストールと設定 (Setup)
スクリプトマネージャーに本コードを追加します。

ローカルでVOICEVOXと、画像リソースをホストするWebサーバーを起動します。

Geminiのページを開くと、右下に起動ボタンが表示されます。

「▶ 実況システム起動」を押すとシステムが展開されます。

各種設定（ウェイクワード、画像URL、音声パラメータなど）は、UI上のボタン（⚙️など）から開ける設定パネルでカスタマイズ可能です。
