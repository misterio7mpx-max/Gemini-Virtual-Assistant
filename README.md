## 概要 (Overview)
ブラウザ上のAIチャット（Gemini）を、音声認識・音声合成・リップシンク（口パク）を備えた「完全なバーチャルアシスタント」へと拡張するTampermonkey向けユーザースクリプトです。
テキストベースのUIを非表示にし、Picture-in-Picture (PiP) を活用した独立アバターウィンドウ上で、ハンズフリーの音声対話を実現します。

## 主な機能 (Features)
* **🗣️ ローカル音声合成連携 (VOICEVOX対応)**
  ローカル環境（`localhost:50021`）のAPIと通信し、AIの生成テキストをリアルタイムに音声キューとして処理・再生します。
* **👄 音声解析によるリアルタイム・リップシンク**
  `AudioContext`と`AnalyserNode`を用いて再生中の音声周波数を解析し、音量に合わせてアバターの口パクを動的に制御します。
* **🖼️ Picture-in-Picture (PiP) によるUIの独立化**
  ブラウザのタブから独立した専用の「アバター・コントロールウィンドウ」を生成し、デスクトップ上に配置可能です。
* **⚙️ 高度なステートマシンとDOM監視**
  `MutationObserver`でAIのストリーミング出力を監視。「待機」「思考中」「発話中」「聞き取り中」などのステータスを管理し、状態に応じたアニメーションやタイピング音を自動で切り替えます。
* **🎤 ハンズフリー音声対話**
  Web Speech APIを利用した音声認識機能を搭載。ウェイクワードによる起動、スリープワードによる待機モードへの移行などに対応しています。

## 技術的なアプローチ (Technical Approach)
**リソース（アセット）の外部化**
初期開発において、アバター画像等のBase64エンコードによるスクリプト内包を試みましたが、データ容量の肥大化によるブラウザのメモリ圧迫を回避するため、アーキテクチャを見直しました。
現在、UI・ロジック（本スクリプト）と画像リソース（外部URL参照）を分離する設計を採用しており、動作の軽量化を図っています。

## 動作環境 (Requirements)
* Tampermonkey などのユーザースクリプトマネージャー
* ローカルで稼働する音声合成エンジン（デフォルト設定は VOICEVOX `localhost:50021` に準拠）
* マイク入力が許可されたブラウザ環境

## インストールと設定 (Setup)
1. スクリプトマネージャーに本コードを追加します。
2. Geminiのページを開くと、画面左側にコントロール用の黒いボタンパネルが表示されます。
3. `⏼` (PiP起動ボタン) を押すと、実況システム（アバターウィンドウ）が独立して立ち上がります。
4. 各種設定（ウェイクワード、画像URL、音声パラメータなど）は、UI上のボタン（①〜⑤）から開ける設定パネルでカスタマイズ可能です。
